Design language board / card table motif

Five ways to extend the home page into the full site.

Each option keeps the current homepage DNA: dark felt, warm playing-card white, black ink, red suits, serif titles, and navigation that feels drawn from a deck. The mockups focus on navbar treatment, page titles, page shells, individual cards, color, type, links, and corner style.

Shared direction

What should stay congruent

Navbar

Keep it compact and game-table inspired: chips, tabs, card labels, or a dealer rail.

Titles

Use a literary serif for page titles so the site still reads as a portfolio, not a game UI.

Pages

Use felt or paper as the page shell. Reserve true card white for clickable content.

Cards

Use rank, suit, rule lines, and subtle rotation as details. Avoid over-decorating every block.

Option 3

Ruby Cut

A sharper, more graphic interpretation. Red becomes the signature accent, corners are square, and the page feels like cut cardstock rather than soft felt.

GREEN INK RUBY CARD
NavbarThin square tabs with a red underline or block.

CardsSharp corners, strong rules, flatter shadows.
Use forMost distinctive option; better if you want less softness.

Option 4

Dealer Grid

A systems-forward version of the table: black felt grid, angled corner cuts, dense cards, and compact nav. It suits technical project pages without losing the deck motif.

GRID GREEN RED CARD
NavbarCompact square-ish tabs with asymmetric clipped-feel corners.
TitlesSerif title plus small monospace-like metadata rows.
CardsDense, slightly angular cards; minimal shadow.
Use forBest for project indexes, case studies, and technical writing.

Option 5

Card Archive

The quietest design language: paper-first pages, modest card corners, black and red suit accents, and a navbar that feels like an index of dealt categories.

PAPER CARD GREEN RED
NavbarSmall rectangular labels, like category dividers.

CardsPlaying-card radius on content; square micro controls.
Use forBest if readability and restraint matter most.
