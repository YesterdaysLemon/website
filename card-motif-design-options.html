Spec sheet / homepage navigation concept

Deck of Cards Motif for Alireza Afshan

The homepage opens with a deck. The deck shuffles, draws five cards, and each card becomes a direct action: Projects, Blog, Resume, About, and Contact. This document gives an implementer six distinct ways to execute that idea while staying compatible with the current React Router site.

Dealer's Table Solitaire Board Quiet Tarot Command Hand Minimal Cut Case File Deck
Recommended baseline

Start with Option 4, Command Hand, if the goal is a portfolio homepage that feels interactive but still professional. Use Option 1 if the shuffle animation should be the main event.

Shared product brief

Core interaction

Navigation Cards

Draw exactly five cards: Projects, Blog, Resume, About, Contact. Each card is a real link. Do not hide normal navigation completely; keep a small fallback nav for accessibility and repeat visits.

Motion Model

Load state shows a compact deck, then 600-900ms of shuffle, then five cards fan or place into final positions. Hover previews lift cards; click navigates. Respect prefers-reduced-motion.

Site Fit

Preserve the current paper/card visual system, serif name treatment, and restrained palette. The motif should feel like a navigational device, not a casino game or novelty landing page.

Six directions

Visual options

1. Dealer's Table

A green felt table gives the deck idea the clearest visual read. The deck sits center-stage, shuffles, then deals a five-card hand across the bottom. Best when you want the homepage to feel memorable immediately.

Interaction Shuffle deck in place, deal cards one by one from center to hand.
Best for A bold homepage with a strong first impression and obvious card metaphor.
Implementation note Use CSS keyframes or Framer Motion. Cards should still render in final state without JS.
Timing: deck appears at 0ms, shuffle rotations 150-650ms, cards deal at 700ms, 820ms, 940ms, 1060ms, 1180ms.

2. Solitaire Board

The homepage behaves like an orderly solitaire layout. The five actions are placed into clean slots, making the motif structured and less theatrical. This direction fits the current minimal site best.

Interaction Cards slide from a draw pile into labeled slots. Hover reveals a short description.
Best for Minimal portfolio navigation where clarity matters more than spectacle.
Implementation note Use CSS grid for the final layout; animate transforms only to avoid layout shift.
Include a persistent "Shuffle again" control, but never require replaying the animation to navigate.

3. Quiet Tarot Spread

A more editorial version of the card idea. The five actions are revealed in an arc, like a spread, with rich labels and subtle symbolic styling. It feels more personal and essay-like than game-like.

Interaction Cards flip from face-down to face-up along an arc, then settle into tappable links.
Best for A distinctive personal site that leans literary, reflective, and atmospheric.
Implementation note Keep text short. Flip animation must not make card labels unreadable on mobile.
Use sparingly: this is the most stylized option and could feel too mystical if pushed too far.

4. Command Hand

The cards are treated as a hand of available commands. One card is focused, and arrow keys or pointer hover can move focus across the hand. This is the strongest option for a software portfolio because it makes the metaphor feel like an interface.

Interaction Initial draw creates a hand. Focused card lifts and shows a preview panel.
Best for Professional, keyboard-friendly navigation with just enough playfulness.
Implementation note Use actual anchor elements inside cards and support tab, arrow keys, enter, and escape.
Recommended choice: it balances the current minimal design with a memorable interaction.

5. Minimal Card Cut

The deck motif is reduced to a simple cut-and-draw gesture. Instead of showing a full hand, the interface lists five card-like rows. This is the safest option if performance, accessibility, and readability outrank spectacle.

Interaction Deck cuts once, then five horizontal card rows slide into view.
Best for Keeping the existing site nearly intact while adding a subtle motif.
Implementation note Works well as pure HTML/CSS with minimal client JavaScript.
This is the easiest direction to implement without creating mobile layout risk.

6. Case File Deck

A hybrid of cards and file tabs. The shuffle reveals five indexed cards that feel like records in a working system. This is less literal, but it connects well to software, systems, and information architecture.

Interaction Cards fan like an index deck; selecting one expands it into a file-style panel.
Best for A mature systems-engineering tone with a light card motif.
Implementation note Map route metadata to card summaries so labels and descriptions stay centralized.
Consider this if the final site should feel like a tool or archive rather than a game.

Build requirements

Routes

Keep existing routes: /projects, /blog, /resume. Add /about if the chosen design uses a dedicated About card; otherwise link About to a homepage section.

Card Data

Create a single array with title, rank, suit, route, summary, and accent. Render the deck and final hand from that data instead of hard-coding each card.

Accessibility

Cards must be keyboard-focusable anchors with visible focus states. Provide skip nav, fallback nav, reduced motion styles, and final-state rendering if JavaScript is disabled.

Motion

Animate only transform and opacity. Avoid layout shifts. Use a deterministic shuffle sequence, not random positions that can overlap text or push cards off-screen.

Responsive

Desktop can show a five-card hand. Tablet can show a shallow fan. Mobile should switch to a two-column or vertical card list so text remains readable.

Visual Tone

Retain the current paper background, serif display type, dark ink, muted copy, and restrained accent colors. Add suits, ranks, and deck backs as details rather than replacing the portfolio tone.

Handoff checklist

Acceptance criteria

Animation complete

On first load, the user sees deck, shuffle, draw, and five stable action cards.

Navigation works

Each card is clickable, keyboard reachable, and routes to the intended page.

No motion trap

Users can navigate immediately after the cards settle, replay is optional, and reduced motion skips the shuffle.

Mobile fit

Card labels, hover/focus states, and controls do not overlap at 320px, 390px, tablet, and desktop widths.

Current content preserved

Existing projects, blog, resume, contact email, and GitHub links remain accessible.

Performance budget

No heavy canvas or 3D dependency unless the chosen design truly requires it. CSS transforms are enough for these concepts.
